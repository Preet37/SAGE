# SocraticTutor Content

Each subdirectory is a self-contained **course package** that the platform
loads automatically on `python seed.py`.

## Directory Structure

```
content/
  <course-slug>/
    course.json        # course manifest (required)
    enrichment/        # reference KBs (optional, generated by Course Creator or manually)
      <lesson-slug>_reference_kb.md
```

## course.json Schema

```json
{
  "slug": "my-course",
  "title": "My Course Title",
  "description": "A short description of the course.",
  "level": "beginner | intermediate | advanced",
  "modules": [
    {
      "title": "Module 1: Getting Started",
      "order_index": 0,
      "lessons": [
        {
          "title": "Introduction",
          "slug": "introduction",
          "order_index": 0,
          "content": "# Introduction\n\nLesson content in Markdown...",
          "summary": "One or two sentences for the tutor system prompt.",
          "concepts": ["concept-a", "concept-b"],
          "youtube_id": null,
          "video_title": null,
          "vimeo_url": null,
          "transcript": null
        }
      ]
    }
  ]
}
```

### Field Reference

| Field | Required | Description |
|-------|----------|-------------|
| `slug` | yes | URL-friendly identifier, must be unique across all courses |
| `title` | yes | Display title |
| `description` | yes | Short course description |
| `level` | no | One of `beginner`, `intermediate`, `advanced` (default: `beginner`) |
| `modules[].title` | yes | Module display title |
| `modules[].order_index` | yes | Sort order within the course |
| `modules[].lessons[].title` | yes | Lesson display title |
| `modules[].lessons[].slug` | yes | URL-friendly identifier, must be unique across all lessons |
| `modules[].lessons[].order_index` | yes | Sort order within the module |
| `modules[].lessons[].content` | yes | Lesson body in Markdown (aim for 2000-4000 chars) |
| `modules[].lessons[].summary` | yes | 1-2 sentence summary used in the tutor system prompt |
| `modules[].lessons[].concepts` | yes | JSON array of key technical terms (4-8 recommended) |
| `modules[].lessons[].youtube_id` | no | YouTube video ID |
| `modules[].lessons[].video_title` | no | Video title |
| `modules[].lessons[].vimeo_url` | no | Vimeo video URL |
| `modules[].lessons[].transcript` | no | Raw video transcript |

## Adding a New Course

1. Create `content/<your-slug>/course.json` following the schema above
2. From `backend/`, run `python seed.py` to load into the database

Alternatively, use the **Course Creator** wizard at `http://localhost:3001` to
build a course interactively — it handles content generation, web-search
enrichment, and publishing to the database in a guided 4-phase workflow.

## Public vs Private Courses

Courses checked into git are public (shared with the repo). To keep a course
private, add its directory to `.gitignore`:

```
content/my-private-course/
```
