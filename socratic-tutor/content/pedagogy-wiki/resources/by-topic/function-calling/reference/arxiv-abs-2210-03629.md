## Introduction

Large Language Models (LLMs) have demonstrated impressive capabilities in diverse tasks, from answering complex questions to generating coherent narratives. However, a fundamental challenge has persisted: how to effectively combine reasoning (the ability to think through problems) with acting (the ability to interact with environments to gather information). The paper "ReAct: Synergizing Reasoning and Acting in Language Models" addresses this gap by introducing a novel paradigm that interleaves these two critical capabilities.
Developed by researchers from Princeton University and Google Research's Brain Team, ReAct draws inspiration from human cognition, where verbal reasoning and action seamlessly integrate during problem-solving. Rather than treating reasoning and acting as separate processes, ReAct enables language models to generate both reasoning traces (thoughts) and task-specific actions in an interleaved manner, creating a synergistic loop that enhances performance across diverse tasks.

## The ReAct Paradigm

At its core, ReAct augments a language model's action space with a "language space" that allows the model to produce intermediate reasoning steps without directly affecting the environment. These reasoning steps serve multiple purposes:

**Planning and tracking progress**: Thoughts help the model formulate, track, and update plans **Handling exceptions**: Reasoning allows the model to identify and recover from errors **Memory and state tracking**: Thoughts create a working memory for complex multi-step tasks **Exploration guidance**: Reasoning guides the exploration of possible actions
The key innovation lies in how reasoning and acting interact: reasoning traces guide the selection of actions, while the results of actions inform subsequent reasoning. This creates a powerful feedback loop that grounds the model's reasoning in factual information obtained through interaction with the environment.

Consider this simplified example of ReAct solving a question:
```

Question: What's the hometown of the author of "The Old Man and the Sea"?

Thought: I need to find who wrote "The Old Man and the Sea" and then determine their hometown.

Action: Search[author of "The Old Man and the Sea"]

Observation: Ernest Hemingway wrote "The Old Man and the Sea" in 1952.

Thought: Now I need to find Ernest Hemingway's hometown.

Action: Search[Ernest Hemingway hometown]

Observation: Ernest Hemingway was born in Oak Park, Illinois.

Thought: The question asks for the hometown of the author. Ernest Hemingway wrote "The Old Man and the Sea" and was born in Oak Park, Illinois.

Answer: Oak Park, Illinois

```

## Foundations in Prior Research

ReAct builds upon two lines of research:

**Reasoning in language models**: Chain-of-Thought (CoT) prompting has shown that LLMs can solve complex reasoning tasks by generating intermediate steps. However, CoT suffers from hallucination and lacks grounding in external knowledge.

**Acting in interactive environments**: Prior work has explored using LLMs for generating actions in interactive settings, but often without leveraging their reasoning capabilities.
ReAct effectively bridges these approaches. Unlike similar work like Inner Monologue, ReAct employs more free-form internal reasoning that helps guide action and systematic exploration.

## Methodology

The researchers implemented ReAct primarily using a frozen (pre-trained) PaLM-540B language model, demonstrating that the approach doesn't require specialized training. Their implementation includes:

**Prompt design**: The model is prompted with a few in-context examples showing ReAct-formatted trajectories (interleaved thoughts and actions)

**Action space definition**: For each task, they define a set of possible actions (e.g., search, answer) that the model can take
**Environment interaction**: An external system processes the model's actions and returns observations

**Trajectory generation**: The model generates thoughts and actions in an alternating sequence until reaching a terminal action

The authors also explored finetuning smaller language models (PaLM-8B and PaLM-62B) on trajectories generated by ReAct, showing that the approach can be adapted to more resource-constrained settings.

## Benchmark Tasks

ReAct was evaluated on four diverse tasks requiring different combinations of reasoning and acting:

**Question Answering (HotpotQA)**: A multi-hop QA dataset requiring reasoning across multiple pieces of information

**Fact Verification (FEVER)**: Determining whether claims are supported, refuted, or unverifiable based on evidence

**Text-based Game (ALFWorld)**: Navigating and completing household tasks in a text-based virtual environment
**Web Shopping (WebShop)**: Finding products on a simulated e-commerce website based on natural language instructions

These tasks were selected to test ReAct's capabilities across different domains, with varying requirements for knowledge retrieval, reasoning, and sequential decision-making.

## Key Results

ReAct demonstrated impressive performance across all benchmarks:

**Knowledge-intensive tasks**: ReAct was competitive with or outperformed Chain-of-Thought approaches on HotpotQA and FEVER, particularly excelling on fact verification where grounding in external knowledge is crucial

**Interactive decision-making**: On ALFWorld and WebShop, ReAct with just one or two in-context examples outperformed imitation learning and reinforcement learning methods trained on many more examples
**Synergistic effects**: Combining ReAct with Chain-of-Thought self-consistency (CoT-SC) led to the best overall performance, showing how internal knowledge and external information can be effectively integrated

**Interpretability**: ReAct provided transparent decision traces that helped humans understand the reasoning behind model actions

**Robust finetuning**: When finetuned on a relatively small dataset of generated trajectories, smaller models showed significant performance gains
A particularly notable finding was that ReAct consistently outperformed acting-only models, demonstrating the value of reasoning to guide actions. ReAct also exceeded the performance of reasoning-only approaches on tasks requiring factual knowledge, showing how interaction with external sources can mitigate hallucination.

## Significance and Impact

The ReAct paradigm represents a significant advancement in language model capabilities with several important implications:

**Human-like problem solving**: ReAct mirrors how humans combine thinking and acting when solving complex problems, leading to more natural and effective AI systems

**Reduced hallucination**: By grounding reasoning in external information, ReAct addresses one of the major limitations of current LLMs - their tendency to generate plausible but incorrect information
**Improved trustworthiness**: The transparent nature of ReAct's reasoning traces allows humans to inspect and potentially correct the model's thought process

**Sample efficiency**: ReAct's strong few-shot performance suggests it can be applied to new tasks with minimal examples, reducing the data requirements compared to specialized training approaches

**Versatility**: The paradigm works across diverse tasks from question answering to interactive navigation, suggesting broad applicability

## Limitations and Future Work

Despite its impressive performance, ReAct has several limitations the authors acknowledge:

**Computational requirements**: Using large models like PaLM-540B requires significant computational resources, though finetuning smaller models shows promise

**Action space design**: The effectiveness of ReAct depends on well-defined action spaces, which may require domain expertise to design
**Error recovery**: While ReAct handles exceptions better than alternatives, it can still get stuck in loops or fail to recover from certain errors

The authors suggest several promising directions for future research:

**Scaling with multi-task training**: Training ReAct across diverse tasks may further improve its generalizability

**Reinforcement learning integration**: Combining ReAct with reinforcement learning could enable optimization beyond imitation
**Human-in-the-loop interaction**: Incorporating human feedback could further improve ReAct's reasoning and action selection

**Application to embodied tasks**: Extending ReAct to physical robots or more complex virtual environments

The ReAct paradigm represents an important step toward more capable, interpretable, and trustworthy AI systems that can effectively reason while interacting with their environment - a fundamental capability for truly useful artificial intelligence.

Recent advances have expanded the applicability of language models (LM) to downstream tasks.
On one hand, existing language models that are properly prompted, via chain-of-thought, demonstrate emergent capabilities that carry out self-conditioned reasoning traces to derive answers from questions, excelling at various arithmetic, commonsense, and symbolic reasoning tasks.
However, with chain-of-thought prompting, a model is not grounded in the external world and uses its own internal representations to generate reasoning traces, limiting its ability to reactively explore and reason or update its knowledge.
On the other hand, recent work uses pre-trained language models for planning and acting in various interactive environments (e.g., text games, web navigation, embodied tasks, robotics), with a focus on mapping text contexts to text actions via the language model’s internal knowledge.
However, they do not reason abstractly about high-level goals or maintain a working memory to support acting over long horizons.
In “ReAct: Synergizing Reasoning and Acting in Language Models”, we propose a general paradigm that combines reasoning and acting advances to enable language models to solve various language reasoning and decision making tasks.
We demonstrate that the *Reason+Act *(ReAct) paradigm systematically outperforms reasoning and acting only paradigms, when prompting bigger language models and fine-tuning smaller language models.
The tight integration of reasoning and acting also presents human-aligned task-solving trajectories that improve interpretability, diagnosability, and controllability..
## Model Overview
ReAct enables language models to generate both verbal reasoning traces and text actions in an interleaved manner.
While actions lead to observation feedback from an external environment (“Env” in the figure below), reasoning traces do not affect the external environment.
Instead, they affect the internal state of the model by reasoning over the context and updating it with useful information to support future reasoning and acting.

We focus on the setup where a frozen language model, PaLM-540B, is prompted with few-shot in-context examples to generate both domain-specific actions (e.g., “search” in question answering, and “go to” in room navigation), and free-form language reasoning traces (e.g., “Now I need to find a cup, and put it on the table”) for task solving.
For tasks where reasoning is of primary importance, we alternate the generation of reasoning traces and actions so that the task-solving trajectory consists of multiple reasoning-action-observation steps.
In contrast, for decision making tasks that potentially involve a large number of actions, reasoning traces only need to appear sparsely in the most relevant positions of a trajectory, so we write prompts with sparse reasoning and let the language model decide the asynchronous occurrence of reasoning traces and actions for itself.
As shown below, there are various types of useful reasoning traces, e.g., decomposing task goals to create action plans, injecting commonsense knowledge relevant to task solving, extracting important parts from observations, tracking task progress while maintaining plan execution, handling exceptions by adjusting action plans, and so on.
The synergy between reasoning and acting allows the model to perform dynamic reasoning to create, maintain, and adjust high-level plans for acting (reason to act), while also interacting with the external environments (e.g., Wikipedia) to incorporate additional information into reasoning (act to reason).
