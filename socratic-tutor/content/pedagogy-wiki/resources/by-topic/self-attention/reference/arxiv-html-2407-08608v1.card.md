# Card: FlashAttention-3 (H100 throughput + FP8 accuracy)
**Source:** https://arxiv.org/html/2407.08608v1  
**Role:** benchmark | **Need:** EMPIRICAL_DATA  
**Anchor:** H100 utilization/TFLOPs/s & speedups; FP8 support and numerical error vs baselines

## Key Content
- **Scaled dot-product attention (Section 2.1):**  
  \(O=\mathrm{softmax}(S)V\), where \(S=\frac{QK^\top}{\sqrt{d}}\).  
  \(Q,K,V\in\mathbb{R}^{n\times d}\) (sequence length \(n\), head dim \(d\)). Softmax applied row-wise. For numerical stability: subtract row max from \(S\) before exp.
- **Why FlashAttention-3 (Intro):** FlashAttention-2 achieves ~**35% utilization** on H100 vs **80–90%** for optimized GEMM; FA-3 redesigns for Hopper **asynchrony** (Tensor Cores + TMA) and **FP8**.
- **Core techniques (Section 3):**
  1) **Warp-specialization producer/consumer** + circular SMEM buffer; use **TMA** for GMEM→SMEM loads and **WGMMA** for async GEMM; **setmaxnreg** reallocates registers to compute warps.  
  2) **Overlap softmax under GEMM** (“pingpong” between warpgroups; plus 2-stage pipelining across iterations) to hide low-throughput exp. H100 example: FP16 matmul **989 TFLOPs** vs special functions (exp) **3.9 TFLOPs**.  
  3) **FP8 forward pass:** FP8 WGMMA requires **k-major** layout; FA-3 uses **in-kernel transpose** via **LDSM/STSM** and register byte-permute to satisfy layout constraints.
- **Empirical performance (Abstract/Section 4):**
  - FP16: **1.5–2.0×** faster than FlashAttention-2 forward; up to **740 TFLOPs/s** (~**75%** of H100 theoretical). Backward: **1.5–1.75×**.  
  - FP8: close to **1.2 PFLOPs/s**.
  - Pingpong scheduling example: **570 → 620–640 TFLOPs/s** (FP16 fwd, \(d=128\), \(n=8192\)).
  - Ablation (Table 2, non-causal FP16): FA-3 **661 TFLOPs/s (3.538 ms)**; no pipelining **582**; no warp-specialization **570**.
- **Numerical error (Section 4.3, Table 3; RMSE vs FP64 ref with outlier Q distribution):**
  - Baseline FP16 **3.2e-4**; FA-2 FP16 **1.9e-4**; FA-3 FP16 **1.9e-4** (≈**1.7×** lower than baseline).  
  - Baseline FP8 (per-tensor scaling) **2.4e-2**; FA-3 FP8 (block quant + incoherent) **9.1e-3** (≈**2.6×** lower error).  
  - Removing incoherent processing returns to **2.4e-2**.
- **Benchmark defaults (Section 4.1/C.1):** H100 80GB SXM5; clock fixed **1830 MHz**; CUDA **12.3**, cuDNN **9.1.1.17**, CUTLASS **3.5**, FA **2.5.8**, Triton nightly **3.0**, PyTorch **2.3**; 100 repeats avg. Seq lens 512…16k; total tokens **16k**; hidden dim **2048**; head dims **64/128/256**.

## When to surface
Use when students ask about (1) exact attention vs “flash” implementations, (2) why softmax is a bottleneck and how to overlap it with GEMM on GPUs, or (3) concrete H100 speed/utilization numbers and FP8 accuracy tradeoffs (block quantization + incoherent processing).
