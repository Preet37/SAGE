# Card: BF16 FlashAttention instability: biased rounding + low-rank error accumulation
**Source:** https://arxiv.org/html/2510.04212v1  
**Role:** paper | **Need:** CONCEPT_EXPLAINER  
**Anchor:** Attention-specific numerical failure modes under BF16 (FlashAttention) and mechanistic chain to loss explosion

## Key Content
- **Failure case (Sec. 3.1):** GPT‑2 pretraining with FlashAttention in **BF16** shows **catastrophic loss explosion after several thousand steps**; reverting to standard attention or FP32 stabilizes but is slower.
- **Repro setup (Sec. 3.1):** GPT‑2 (12 layers, 12 heads, d_model=768, ctx=1024), OpenWebText. AdamW: β1=0.9, β2=0.95, **weight decay=0**, cosine LR with **2000 warmup** to peak **6e‑4**, decay to **6e‑5**; **global grad clip=1.0**. 4×A100‑80GB, AMP: **BF16 forward, FP32 backward**. Microbatch 32/GPU, grad accum 4 ⇒ **524,288 tokens/step**.
- **Isolation (Sec. 3.2):**
  - Tiling not culprit: set block size = sequence length; still fails.
  - Failure originates in **layer 2 attention**; using FA only in layer 2 reproduces; replacing FA with standard attention in layer 2 restores stability.
  - Failure tied to backward term **\(D\)** computed from low‑precision output **\(O_{lp}\)**; recomputing **\(O\)** in FP32 or computing **\(O\)** in FP32 in forward stabilizes.
  - Localized to a few heads; computing **\(O\)** in FP32 for heads **{1,7,8,9,11,12}** stabilizes (head 8 worst).
- **Gradient error mechanism (Sec. 3.3.1):**
  - Query grad error: **Eq. (1)**  
    \(\Delta(\nabla Q)=\nabla Q_{hp}-\nabla Q_{lp}=\mathrm{diag}(\Delta D)\,(P V)\,\alpha\)  
    where \(P=\mathrm{softmax}(S)\), \(S=QK^\top/\sqrt{d}\), \(V\)=values, \(\alpha\)=attention scaling factor, \(D\)=key backward intermediate, \(\Delta D=D_{hp}-D_{lp}\).
  - Weight grad error for \(W_Q\): **Eq. (2)**  
    \(\Delta(\nabla W_Q)=\sum_i (\Delta D_i)\,(X_i^\top \otimes (PV)_i)\) (rank‑1 terms).
  - Similar rank‑1 structures across steps/tokens ⇒ approx **Eq. (3)**  
    \(\Delta(\nabla W_Q)\approx \left(\sum_i \Delta D_i\right) R\). Empirically \(\sum_i \Delta D_i\) stays **consistently positive** (steps 6580–6680), so error accumulates → spectral norm spike → divergence.
- **Numerical root cause (Sec. 3.3.2):**
  - Bias comes from BF16 accumulation in unnormalized output **\(O' = P V\)**; computing only this product in FP32 stabilizes.
  - Large negative error steps occur when some attention probs are **exactly 1** (row max in scores ⇒ exp(0)=1). With \(P_{ij}=1\), product term equals \(V_j\) (often **predominantly negative** in failing dims), and **adding negative BF16 numbers** can overflow significand → right shift + rounding; “round‑to‑nearest ties‑to‑even” becomes **systematically more negative** in this distribution, creating biased \(O_{lp}\) and thus biased \(D\).
- **Mitigation (Sec. 4, App. C):** Modify safe softmax **only when a row has multiple identical maxima**; adjust row-wise normalization constant so max exponent is **strictly negative**, guaranteeing **all \(P<1\)**. Uses softmax shift-invariance: \(\mathrm{softmax}(x)=\mathrm{softmax}(x+c)\). Parameter: **\(\epsilon=2^{-7}\)** (smaller may round back to 1; larger risks underflow). Fixed offsets are discouraged (introduce new systematic rounding bias).

## When to surface
Use when students ask why scaled dot-product attention/FlashAttention becomes unstable in BF16/FP16, how softmax “probability = 1” and accumulation/rounding interact, or what principled numerical fixes (beyond ad-hoc clipping) look like.
