# Card: Scaled Dot-Product Attention (Transformer)
**Source:** https://arxiv.org/html/1706.03762v7  
**Role:** paper | **Need:** FORMULA_SOURCE  
**Anchor:** Section 3.2.1 Scaled Dot-Product Attention formula + scaling rationale (variance/softmax saturation)

## Key Content
- **Scaled Dot-Product Attention (Section 3.2.1, Eq. 1):**  
  \[
  \mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
  \]
  - \(Q\): queries matrix (packed set of queries), key dimension \(d_k\)  
  - \(K\): keys matrix, same key dimension \(d_k\)  
  - \(V\): values matrix, value dimension \(d_v\)  
  - Output: weighted sum of values; weights from softmax over attention logits \(QK^\top/\sqrt{d_k}\).
- **Rationale for scaling by \(1/\sqrt{d_k}\) (Section 3.2.1):**
  - For large \(d_k\), dot products \(q\cdot k\) grow in magnitude, pushing softmax into saturated regions with **extremely small gradients**.
  - Variance argument: if components of \(q\) and \(k\) are i.i.d. with mean 0 and variance 1, then \(q\cdot k\) has mean 0 and variance \(d_k\). Scaling by \(1/\sqrt{d_k}\) counteracts this.
- **Masking in decoder self-attention (Section 3.2.3):**
  - To preserve autoregression, **mask out illegal connections** by setting corresponding softmax inputs (logits) to \(-\infty\) before softmax (prevents attending to future positions).
- **Dot-product vs additive attention (Section 3.2.1):**
  - Dot-product attention is faster/more space-efficient (matrix multiplication); additive can outperform unscaled dot-product for large \(d_k\), motivating scaling.

## When to surface
Use when students ask how attention is computed (logits → softmax → weighted sum), why the \(\frac{1}{\sqrt{d_k}}\) factor exists, or how causal masking is implemented inside attention.
