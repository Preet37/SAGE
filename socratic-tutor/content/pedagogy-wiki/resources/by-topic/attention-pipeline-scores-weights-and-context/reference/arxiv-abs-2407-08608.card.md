# Card: FlashAttention-3 benchmarks + low-precision numerical notes
**Source:** https://arxiv.org/abs/2407.08608  
**Role:** benchmark | **Need:** EMPIRICAL_DATA  
**Anchor:** Empirical results on attention speed/accuracy (FP16/FP8) + numerical considerations in fused attention kernels

## Key Content
- **Scaled dot-product attention (Sec. 2.1):** For one head with sequence length \(n\), head dim \(d\):  
  \[
  O=\mathrm{softmax}(S)V,\quad S=\alpha QK^\top,\ \alpha=\frac{1}{\sqrt{d}}
  \]
  Softmax applied row-wise; for numerical stability subtract row max: \(S \leftarrow S-\max(S)\) before exp.
- **Why fused/Flash attention is more numerically accurate (Sec. 4.3):** FP16 FlashAttention-2/3 keep intermediate softmax/rescaling in **FP32**, yielding lower error than “standard attention” that materializes intermediates.
- **Empirical speed (Abstract, Sec. 4.1):**
  - On **H100 SXM5**, FlashAttention-3 achieves **1.5–2.0×** forward speedup vs FlashAttention-2 (FP16), reaching **up to 740 TFLOPs/s (~75% utilization)**.
  - Backward pass: **1.5–1.75×** faster vs FlashAttention-2.
  - Versus standard attention: **~3–16×** faster.
  - FP8 forward: **close to 1.2 PFLOPs/s**.
- **Ablation (Table 2, FP16 non-causal):**  
  FlashAttn-3 **3.538 ms, 661 TFLOPs/s**;  
  No GEMM-softmax pipelining (but warp-specialization) **4.021 ms, 582**;  
  No warp-specialization (but pipelining) **4.105 ms, 570**.
- **Numerical error (Table 3, vs FP64 reference; outlier Q distribution: 0.1% entries add \(\mathcal{N}(0,10)\) to \(\mathcal{N}(0,1)\)):**
  - RMSE: Baseline FP16 **3.2e-4**; FlashAttn-2 FP16 **1.9e-4**; FlashAttn-3 FP16 **1.9e-4** (**1.7× lower** than baseline).
  - FP8: Baseline per-tensor scaling RMSE **2.4e-2**; FlashAttn-3 FP8 (block quant + incoherent) **9.1e-3** (**2.6× lower**).
- **Benchmark defaults (Sec. 4.1):** seq len \(=512,1k,\dots,16k\); batch size chosen so total tokens **16k**; hidden dim **2048**; head dim **64/128/256** (32/16/8 heads). Causal masking halves FLOPs. Backward FLOPs ≈ **2.5×** forward.
- **Design rationale (Sec. 3):** overlap softmax (exp throughput ~**3.9 TFLOPs** special functions) under async Tensor Core GEMMs (H100 FP16 matmul ~**989 TFLOPs**) via warp-specialization + pipelining; FP8 needs block quantization + incoherent processing to reduce outlier-driven quantization error.

## When to surface
Use when students ask for **concrete benchmark numbers** for FlashAttention vs standard attention, or why **FP16 fused attention can be more accurate** (FP32 softmax intermediates) and how **FP8 attention** mitigates quantization error (block quantization + incoherent processing).
