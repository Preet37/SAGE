Every LLM can explain attention mechanisms.

Answering ≠ Teaching

What ChatGPT does

  • Dumps the correct answer immediately
  • No awareness of what you already know
  • Cites training data — or hallucinates URLs
  • No curriculum — every question is standalone

What a tutor does

  • Probes understanding before explaining
  • Adapts to gaps in your knowledge
  • Grounds answers in curated, verified sources
  • Tracks progress across a full curriculum

An AI-first tutor is not a chatbot with a textbook —
it's a pipeline of specialized AI systems.

Five Design Decisions

If you were building this, what would you do?

Decision 1

Where does knowledge come from?

"Your tutor needs to explain LoRA. Do you…"

(a) LLM training data

Fast. But unverifiable. May hallucinate citations. Stale by months.

(b) RAG over papers

Grounded. But noisy retrieval. Teaching quality depends on what you indexed.

(c) Something else

…

Decision 1 — Wiki-First Architecture

A curated knowledge base organized for teaching, not just retrieval.

Two tracks per topic

  • Pedagogy track — how to teach it (analogies, visuals, common misconceptions)
  • Reference track — precision facts (formulas, benchmarks, API details)

Why it matters

  • Citation validation catches hallucinated URLs
  • Wiki grows with every new course created
  • Sources are curated by an LLM curator + reviewer pipeline

The enrichment step (stages 3-4)

  • LLM assesses concept-level coverage: covered / thin / missing
  • Generates targeted search queries for gaps
  • Curates best sources (curator + reviewer audit loop)
  • Downloads, indexes, extracts reference cards

Multimodal tie-in

  • Image pipeline extracts diagrams from HTML sources
  • Vision LLM annotates each image: caption, concepts, teaching value, "when to show"
  • Cross-topic deduplication saved 430 MB

Decision 3

How do you make it actually teach?

"You have great content. But the LLM still just… answers. How do you make it Socratic?"

Decision 3 — Two-Layer Knowledge

Student Notes (what learners read)

  • Narrative-driven, varied openings
  • Analogies when they clarify, not forced
  • LaTeX math, code examples, comparison tables
  • Recommended reading at the end

Reference KB (what the tutor consults)

  • Teaching approaches (intuitive / technical / analogy)
  • Common misconceptions with corrections
  • Socratic question bank
  • "Show when" guidance for visuals and resources
  • Key formulas and empirical results

Tutor toolkit — 7 tools

search_web get_lesson_context get_relevant_images get_curated_resources get_lesson_reference_kb get_lesson_transcript search_curriculum

The LLM decides when to use each tool based on the conversation context — not hardcoded rules.

System prompt enforces Socratic behavior: "Probe understanding before explaining. Ask a question before giving an answer."

Decision 5 — How do you know it works?

"You ship it. How do you know students are actually learning?"

Tier 1: Structural

Required sections, word counts, LaTeX balance, URL validity. Runs in <1 second.

Tier 2: Grounding

Citation validation against wiki. Concept coverage vs. outline. Source diversity.

Tier 3: LLM Judge

Rubric scoring on accuracy, engagement, pedagogy. Format-probe scenarios.

Format compliance checks

Quiz cards used? Resource cards have URLs? Responses aren't getting shorter? Duplicates caught?

Regression-catching scenarios

Automated multi-turn conversations that probe specific format behaviors — catches prompt regressions before deploy.
